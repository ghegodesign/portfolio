### Node (Node.js)

* **Cos’è**: un runtime che ti permette di eseguire **JavaScript fuori dal browser**.
* **A cosa serve**:

  * backend (server, API)
  * tool di build (Vite, Webpack, npm scripts, ecc.)
  * CLI e automazioni
* **Come funziona**:

  * usa il motore **V8 di Chrome**
  * è event-driven, non bloccante
* **Fatto chiave**: senza Node **non esiste l’ecosistema moderno JS** (npm, bundler, framework).

---

### nvm (Node Version Manager)

* **Cos’è**: un **gestore di versioni di Node**.
* **A cosa serve**:

  * installare **più versioni di Node** sullo stesso computer
  * passare da una versione all’altra senza casino
* **Perché è indispensabile**:

  * progetti diversi richiedono versioni diverse di Node
  * evita rotture, incompatibilità, bug stupidi
* **Esempio pratico**:

  * progetto A → Node 16
  * progetto B → Node 20
  * con nvm: `nvm use 16` / `nvm use 20`

---

### In una frase

* **Node** = esegue JavaScript.
* **nvm** = decide *quale* Node stai usando.

Se vuoi, ti dico **quando serve davvero installarli**, o **come capire che versione di Node ti chiede un progetto**.
